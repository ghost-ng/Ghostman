Getting Started

Installation

Requirements: An OpenAI API key or compatible AI service

  1. Download the latest ghost-ng installer from GitHub Releases
  2. Run the installer and follow the setup wizard
  3. Launch ghost-ng from your desktop shortcut or Start menu
  4. Configure your API key in Settings (see API Setup below)

First Run

  1. Launch ghost-ng from your desktop or command line
  2. The application will start in system tray mode
  3. Right-click the tray icon and select "Show Avatar" to open the main interface
  4. Click the gear icon to open settings and configure your API key

API Setup

‚ö†Ô∏è Important: You need an AI service API key to use ghost-ng's features.

OpenAI Setup (Recommended)

  1. Visit OpenAI Platform
  2. Create an account or sign in
  3. Go to API Keys section and create a new key
  4. Copy the API key
  5. In ghost-ng, click the gear icon ‚Üí AI Model
  6. Paste your API key in the API Key field
  7. Click "Test Connection" to verify

OpenAI-Compatible Services

ghost-ng also works with OpenAI-compatible API providers:

  • Anthropic Claude: Use with proxy services
  • Local AI: Ollama, LM Studio, or similar
  • Azure OpenAI: Enterprise deployments
  • Custom endpoints: Any OpenAI-compatible API

Configure the Base URL and API key in Settings ‚Üí AI Model for alternative providers.

Troubleshooting

Frequently Asked Questions

‚ùì ghost-ng won't start or crashes immediately

Try these solutions:

  • Restart the application
  • Check if you have the latest version installed
  • Try running as administrator (Windows only)
  • Check logs in: %APPDATA%/Ghostman/logs/ (Windows)

‚ùì API connection fails

Common API issues:

  • Verify your OpenAI API key is valid and has credits
  • Check internet connectivity
  • Ensure firewall isn't blocking the connection
  • Try a different model (GPT-3.5-turbo vs GPT-4)

‚ùì Conversations not saving

Database troubleshooting:

  • Check if you have write permissions to the app data folder
  • Restart the application to reinitialize the database
  • In Settings ‚Üí Advanced, try "Reset Database"
  • Check available disk space

Log Files

Log files can help diagnose issues:

Log Locations

Windows: %APPDATA%\Ghostman\logs\
macOS: ~/Library/Application Support/Ghostman/logs/
Linux: ~/.config/Ghostman/logs/
